## 1. 项目是做什么的

纯前端 **对码审核工作台高保真 UI Demo**：左侧为 **一条当前待处理的美团商品资料**（唯一参考），右侧为 **多条待匹配的 DrugID 商品资料**。审核员在右侧对比候选，做出匹配决策并进入下一条；另有 **匹配历史** 独立页面做查询、分 Tab 浏览与导出。

当前无后端：数据为脚本内 **`buildDefaultTasks()`** 生成的演示队列 + **`localStorage`** 持久化。

## 4. 工作台（`index.html`）功能要点

### 4.0 视觉方向

- 当前已进入 UI 设计阶段：参考「柔和蓝灰背景 + 超大圆角容器 + 玻璃拟态面板 + `#0088FF` 商用蓝主按钮 + Bento 卡片」风格。
- 高保真视觉主要通过两页底部的 **High fidelity UI skin** CSS 覆盖实现，不改变现有业务结构和交互逻辑。

### 4.1 布局

- 页面外层加入 **O2On 商品治理系统侧边栏**，当前菜单为 **对码审核**，并提供 **匹配历史** 跳转。
- **左**：当前任务的美团商品（名称、规格、条形码/UPC、批准文号、生产厂家、处方分类、最多 6 张图）。
- **中**：当前任务下 **最多 4 个** Drug 分区并排对比；**满 4 区时为 2×2 田字格**，非四区时为横向 flex 等分。
- **右**：书签栏，列出 **当前任务** 的全部候选 Drug；支持 **仅展示 / 加入展示 / 退出展示**；正在展示的书签 **向左突出** 并高亮。
- Drug 分区与书签统一展示为 **`【drugid】商品名称 + 置信度标签`**；置信度三档为 **高置信度 / 中置信度 / 低置信度**，字段为 `confidence: high|medium|low`，未传时前端会根据候选 ID 做默认展示。
- **图层**：书签栏 `z-index` 高于中间商品卡，避免被遮挡；`.main` 使用 `overflow: visible` 以免突出书签被裁切。

### 4.2 处方与批准文号

- 所有商品（美团 + Drug）均包含 **批准文号**。
- **处方分类** 仅三类：**处方 / 非处方 / 其他**（字段 `rxCategory`）。

### 4.3 匹配交互

- 每个可见 Drug 分区顶部有 **「选中为匹配商品」** 复选框；**全局最多勾选 1 个**（互斥逻辑在脚本中实现）。
- **「匹配成功，下一条」**：必须先勾选一条 Drug；将当前任务记为 `matched`，写入 `matchedDrugId`、`matchedDrugName`、`resolvedAt`，再进入下一任务。
- **「需新建 DrugID，下一条」**：不要求勾选；将当前任务记为 `new_drug`，清空匹配 Drug 字段，记录 `resolvedAt`，再进入下一任务。
- 全部任务处理完后：隐藏工作台，展示 **「您已完成所有审核匹配任务，辛苦了」**。

### 4.4 顶部进度条

- 展示「还剩 N 条待匹配」及「已完成 x / 共 y」；进度条长度为已完成比例；全部完成后为 100%。

### 4.5 任务队列（演示数据）

- 脚本内 **`buildDefaultTasks()`** 定义 **5 条任务**（`task-1` … `task-5`），每条含 `meituan` + `candidates[]`。
- 每条候选有独立 `id`（如 `t1-c1`），避免跨任务冲突。

## 5. 匹配历史页（`history.html`）

- 页面外层加入 **O2On 商品治理系统侧边栏**，当前菜单为 **匹配历史**，并提供 **对码审核** 返回。
- **Tab 在筛选区上方**：切换 Tab 后，筛选项与列表字段按状态动态变化。
- **待匹配筛选**：商品名称、美团 UPC。
- **匹配成功筛选**：商品名称、美团 UPC、匹配成功的 DrugID、匹配时间范围。
- **待新建 DrugID 筛选**：商品名称、美团 UPC、匹配时间范围。
- **列表基础字段**：商品名称、商品规格、商品条码、批准文号、处方分类、生产厂家。
- **匹配成功列表额外字段**：匹配成功的 DrugID、匹配时间。
- **待新建 DrugID 列表额外字段**：匹配时间。
- **分页组件**：列表底部提供通用分页，当前每页 `10` 条，支持上一页 / 下一页 / 页码跳转。
- **导出**：导出当前 Tab + 当前筛选结果为 CSV；CSV 包含列表字段，并额外带当前状态、匹配成功的 DrugID、匹配时间，便于线下复核。
- **待匹配 + 时间筛选**：无 `resolvedAt` 的待匹配行 **不参与时间条件过滤**（避免被时间筛空），见 `history.html` 内 `inTimeRange` 注释。

## 6. 本地存储（接接口前必读）

| 键名 | 用途 |
|------|------|
| `meituan_drug_match_workbench_v2` | **唯一主状态**：`version: 2`、`tasks[]`、`currentTaskIndex`、`visibleDrugIds`、`selectedDrugId` |
| `meituan_drug_match_demo_state` | **旧版遗留键**；`index.html` 若读不到 v2 会尝试忽略/不再依赖，勿与新逻辑混用 |

**任务对象（`tasks[]` 中每一项）关键字段：**

- `id`：任务 ID  
- `status`：`pending` \| `matched` \| `new_drug`  
- `meituan`：`name, spec, barcode, approvalNo, manufacturer, rxCategory, images[]`  
- `candidates[]`：每条含 `id, drugId, name, spec, barcode, approvalNo, manufacturer, rxCategory, images[]`  
- `matchedDrugId` / `matchedDrugName` / `resolvedAt`：在 `matched` 或 `new_drug` 时按代码写入  

**归一化逻辑**：`normalizeState()` 会把 `currentTaskIndex` 校正到 **第一个 `pending` 任务**；若全部已处理则索引为 `tasks.length`，进入完成态。

## 8. 已知的实现边界（便于后续迭代）

- 无登录、无接口；队列与候选写死在 `buildDefaultTasks()`。
- 勾选为 **checkbox + 脚本互斥**，非原生 `radio`（若改无障碍可再评估）。
- 小屏下书签为横向滚动，突出样式与桌面略有差异（媒体查询内已处理）。
- 历史页依赖本机 `localStorage`，清浏览器数据会丢记录。
